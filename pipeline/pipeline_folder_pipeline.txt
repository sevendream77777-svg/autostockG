F:\autostockG\pipeline 폴더 자체 파이프라인 분석
====================================================

이 문서는 pipeline 폴더 내부 구조 자체를 설명하기 위한 전용 파이프라인 파일입니다.

[역할 요약]
- 이 폴더는 전체 프로젝트의 “문서 자동 생성 시스템”을 담당.
- HOJ/SLE 엔진 실행 로직과 완전히 분리된 정리·분류·문서화 전용 구역.
- 루트 코드와 DB를 스캔하고 관계를 요약하여 정적 문서로 만드는 것을 중심 목적.

[구성(SNAPSHOT)]
1) auto_generate_pipeline_doc.py
   - 전체 파이프라인 보고서를 자동 생성한다.
   - 같은 날짜 동명 파일 존재 시 자동으로 -1, -2 번호 증가 생성.

2) autopipeline_mapper.py
   - py 파일 import 구조, 경로 접근 패턴 파악.

3) pipeline_auto_analyzer.py
   - mapper 결과를 기반으로 종합 분석 및 섹션 구조화.

4) scan_path_usage.py
   - root 전체에서 절대경로/상대경로 하드코딩된 부분 스캔.

5) detect_locked_file.py
   - Windows에서 파일 잠김 감지.

6) config_paths.py (선택)
   - 전체 경로 설정을 모아둔 파일.

7) run_pipeline_v6.py (구버전 통합 파이프라인)
   - 참고용. 문서 생성 자동화 흐름 설계용.

8) patch_engine / patch_paths_no_backup / apply_auto_patch
   - 경로/엔진 패치 관련 보조 도구. 문서 자동화 범주에 포함 가능.

[동작 흐름]
pipeline 버튼/UI → run_pipeline_doc.bat → auto_generate_pipeline_doc.py
→ 루트 전체 스캔 → pipeline_YYYYMMDD[-N].txt 생성.
